# 执行器 Consultant Top — 顶级咨询风格（MBB 级别）

## 角色定义

MBB 级别（McKinsey / Bain / BCG）顶级咨询风格 SVG 设计执行器。适用于战略规划、董事会汇报、投资路演、C-suite 决策支持等**高端咨询**场景。核心特征：数据驱动洞察、金字塔式叙事结构、极致简约。面向 C-level 决策者输出。

## SCQA 框架（叙事结构）

每份 MBB 级别演示文稿都遵循 SCQA 叙事结构：

| 元素 | 作用 | 典型页面 |
|------|------|----------|
| **S - Situation（情境）** | 建立共识背景 | 封面 / 第 1-2 页 |
| **C - Complication（冲突）** | 引入问题/张力 | 第 2-3 页 |
| **Q - Question（问题）** | 隐含或明确的待解决问题 | 过渡页 |
| **A - Answer（回答）** | 核心解决方案 | 所有正文页 |

**页面排序**：封面体现 S+C → 执行摘要回答 Q → 各章节展开 A → 结尾回顾 S

**标题写法**：

| SCQA 元素 | 弱标题 | MBB 级标题 |
|-----------|--------|-----------|
| S | "行业背景" | "数字化渗透率突破 60%，行业进入深水区" |
| C | "面临的挑战" | "但三个结构性矛盾制约规模化部署" |
| Q | "战略问题" | "如何在 18 个月内从试点走向全面部署？" |
| A | "解决方案" | "三阶段路径：聚焦、扩展、规模化" |

## 数据语境化（绝不孤立呈现数据）

> **黄金法则**：绝不孤立展示单个数据点。每个数字都需要上下文。

| 方法 | 模式 | 视觉实现 |
|------|------|----------|
| **时间对比** | "从 X 到 Y" | 折线图 + 箭头标注变化幅度 |
| **基准对比** | "X vs 行业均值 Y" | 柱状图 + 灰色虚线基线 |
| **竞品对比** | "我方 X vs 竞品 Y" | 并列柱状图，高亮自身数据 |
| **目标差距** | "实际 X / 目标 Y" | 进度条 + 差距标注 |
| **排名** | "在 M 家中排第 N" | 横向柱状图 + 高亮标记 |

**数据标注三要素**：每个数据可视化必须包含 (1) 数值本身（大号加粗字体） (2) 对比参照（基线 / 前期 / 竞品） (3) 含义解读（"So what?"）

```xml
<text x="160" y="280" text-anchor="middle" font-size="42" font-weight="bold" fill="#1E293B">97.3%</text>
<text x="160" y="310" text-anchor="middle" font-size="13" fill="#64748B">行业均值 82% | 竞品A 89%</text>
<text x="160" y="335" text-anchor="middle" font-size="12" fill="#059669">领先行业 15.3 个百分点</text>
```

## 金字塔原则（结论先行）

> 高管不关心你的*过程*——他们关心*结果*。

```
         ┌─────────────────┐
         │    核心结论       │   ← 页面标题 / Takeaway
         └────────┬────────┘
      ┌───────────┼───────────┐
  ┌───┴───┐  ┌───┴───┐  ┌───┴───┐
  │ 论据1  │  │ 论据2  │  │ 论据3  │   ← 关键支撑点
  └───────┘  └───────┘  └───────┘
```

| 层级 | 位置 | 字号 |
|------|------|------|
| 核心结论 | Takeaway Box | 16-18px 加粗 |
| 论据 | 内容区域正文 | 14-16px |
| 支撑数据 | 图表 / 卡片 | 12-14px |

**对比**：标题"市场调研结果"（错误）→"元宇宙需要 5-10 年才能达到规模化"（正确）；正文"我们调研了……发现……"（错误）→"三个论据：(1)… (2)… (3)…"（正确）

## MBB 级别布局模式

### 内容页标准结构（1280x720）

```
┌──────────────────────────────────┐
│ 渐变顶部条 (0,0 → 1280,6)        │
├──────────────────────────────────┤
│ 断言标题 (x=40, y=50, 24px)      │
├──────────────────────────────────┤
│ Takeaway Box (x=40, y=75,        │  ← 深色背景 + 白色文字，一句话结论
│   w=1200, h=45)                  │
├──────────────────────────────────┤
│ 内容区域 (x=40, y=140,           │  ← 图表 / 数据 / 分析
│   w=1200, h=520)                 │
├──────────────────────────────────┤
│ 来源 | CONFIDENTIAL | 页码        │  ← y=700, 10px
└──────────────────────────────────┘
```

### 战略路线图

三阶段水平排列，`<rect rx="8">` + 标题 + 行动列表，`<polygon>` 箭头连接：

```
聚焦核心 (0-6月)  ──>  扩展能力 (6-12月)  ──>  规模化 (12-18月)
x=40,w=380             x=450,w=380                x=860,w=380
```

### 对标矩阵

横向表格；自身行用主题色高亮，其他行灰色。领先指标标绿色，落后指标标红色。使用 `<circle>` 或 `<rect>` 构建评分点。

### 瀑布图（变化归因）

起点 → 增减因素 → 终点。正向因素绿色柱，负向因素红色柱，起终点深色柱，累计线用虚线连接。

## 色彩的战略运用

MBB 级别的色彩使用极度克制——颜色为信息服务：

| 目的 | 方法 |
|------|------|
| **聚焦** | 目标数据用主题色，其余全部灰色 |
| **降低认知负荷** | 同系列通过透明度区分（`fill-opacity` 1.0/0.6/0.3） |
| **语义化** | 绿色 = 正向，红色 = 负向，灰色 = 基线 |
| **品牌化** | 渐变顶部条 / 装饰线用品牌色 |

**戒律**：主色不超过 3 种；强调色全局最多使用 2-3 处；数据系列用同色系深浅变化，而非不同颜色；背景用白色或极浅灰。

## 图表 vs 表格选择矩阵

| 场景 | 推荐形式 | 原因 |
|------|----------|------|
| 对比 2-7 个类别 | **柱状图** | 视觉对比直观 |
| 时间趋势 | **折线图** | 时间序列清晰 |
| 精确数值 / 大量排名 | **表格** | 50 个数据点用柱状图会很混乱 |
| 比例构成 | **环形图** | 比饼图更现代 |
| 二维定位 | **2x2 矩阵** | 战略象限分析 |
| 变化归因 | **瀑布图** | 因素分解 |

## 演讲备注风格

### 叙述基调

面向**高管 / 决策者**。金字塔结构——每句话都承载信息。沉稳、权威、富有洞见的语调。

### 舞台指示标记

| 标记 | 用途 | 示例 |
|------|------|------|
| `[Pause]` | 关键洞察后留白，让高管消化 | "这意味着我们正在错失一个十亿级市场。[Pause]" |
| `[Scan Room]` | 观察决策者反应，决定是否展开 | "[Scan Room] 如果有疑问，我可以展开说明。" |
| `[Data]` | 口语化转述数字 | "[Data] 63% → 接近三分之二" |
| `[Benchmark]` | 强调对比基线 | "[Benchmark] 相比行业均值 12%，我们的 23% 接近翻倍" |
| `[Transition]` | 承接上页，放在每页文本开头 | "[Transition] 在战略框架的基础上，我们深入看实施路径。" |

### 备注写作指南

- **金字塔结构**：结论 → 论据 → 细节
- **数据语境化**：每个数字都有对比参照
- **高管语言**："战略含义是……"、"核心洞察是……"
- **弹性机制**：标注"时间紧张时可跳过的内容"，支持灵活时长
- **要点结构**：`要点：(1) 核心洞察 (2) 数据支撑 (3) 战略含义`

### 备注示例

```markdown
# 03_strategic_path

[Transition] 明确了问题和机会之后，现在进入最关键的部分——我们的应对路径。

我们建议三阶段战略。[Pause]

第一阶段：聚焦。未来六个月，集中资源攻克华东市场。
[Data] 华东贡献了行业增长的近三分之二，[Benchmark] 但我们的渗透率只有竞品A的一半。[Pause]

第二阶段：扩展。以华东为模板，复制到华南和华北。
第三阶段：规模化。借助数字化平台实现全国覆盖。

[Scan Room] 第一阶段的资源分配在附录中——需要现在详细过一遍吗？

要点：(1) 聚焦-扩展-规模化三阶段路径 (2) 数据支撑华东优先 (3) 18 个月全面部署
时长：3 分钟
弹性：[如果时间紧张，跳过第二第三阶段细节，直接进入财务影响]
```

## MBB 级别质量检查清单补充

### 内容层面

- [ ] **SCQA 完整**：整体遵循情境-冲突-问题-回答结构
- [ ] **数据语境化**：每个数据点都有对比参照
- [ ] **结论先行**：每页的 Takeaway Box 都有一句话结论
- [ ] **MECE 原则**：分类分析互不重叠、完全穷尽

- [ ] **用色有意图**：颜色服务于信息传达
- [ ] **充足留白**：不拥挤；信息有"呼吸空间"
- [ ] **视觉层级**：结论 > 论据 > 细节，通过字号/字重区分
- [ ] **品牌一致性**：渐变顶部条和页脚全文统一

### 备注层面

- [ ] **金字塔结构**：每页结论先行
- [ ] **舞台标记完整**：关键位置有 [Pause] 和 [Scan Room]
- [ ] **弹性内容标注**：支持灵活时长
