# 设计哲学库

> 为 PPT/SVG 场景优化的 14 种设计哲学精选集。在策略师进行风格确认时，用作设计方向推荐的参考（详见 `strategist.md` § 设计哲学推荐）。

**使用方法**：当用户的风格偏好模糊或寻求差异化时，从不同流派中推荐 3 种哲学。用户选定后，将该哲学的 **Visual DNA** 注入到 `design_spec.md` 的 Mood Statement 和视觉约束中。

---

## 流派一：信息架构（理性·数据·克制）

### 01. Pentagram — 信息架构

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 极致的字体层次结构配合瑞士网格纪律。60%+ 留白比例。标题文字以 3-4 倍正文大小主导整页。每个元素的放置都追求几何精度。 |
| **SVG 转化** | 严格基于网格的 x/y 坐标；标题 `font-size` ≥ 60px 且 `font-weight="bold"`；正文克制在 18-20px；宽裕的边距（80px+）；极少装饰元素；单色调色板加单一强调色 |
| **代表作品** | Pentagram 品牌识别系统、Bloomberg Businessweek 编辑版面 |
| **PPT 最佳用途** | 战略报告、董事会演示、数据密集的高管幻灯片 |
| **色彩策略** | 近单色：黑 + 白 + 一个机构强调色（海军蓝、酒红色或森林绿） |
| **适配性** | ★★★★★ — 最自然契合专业演示的哲学 |

### 02. Fathom — 数据叙事

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 读起来像编辑新闻的数据可视化。图表融入叙事流中，不是孤立在方框里。注释直接标注在数据点上。为数字赋予"新闻编辑室"美学。 |
| **SVG 转化** | 数据可视化元素（柱形、折线、面积）配合内联 `<text>` 注释标注在关键数据点上；引线连接数据与洞察；柔和的网格线（`stroke-opacity="0.1"`）；最重要的单一数据点用粗体强调色突出 |
| **代表作品** | 纽约时报数据新闻、The Pudding 交互故事、Fathom Information Design |
| **PPT 最佳用途** | 数据分析报告、财务复盘、市场调研演示 |
| **色彩策略** | 中性底色（暖灰或米白）+ 数据强调色层级（主要指标用粗体色彩，次要指标用柔和色调） |
| **适配性** | ★★★★★ — 完美适配咨询和数据类幻灯片 |

### 03. Müller-Brockmann — 瑞士网格

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 数学网格系统作为所有视觉决策的基础。非对称但和谐的版面。字体排印是首要设计元素。客观的视觉传达。 |
| **SVG 转化** | 所有元素吸附到 12 列网格（列宽 = (内容宽度 - 11×间距) / 12）；统一使用 `text-anchor="start"` 的左对齐层次；严格限制调色板（最多 2-3 色）；无装饰元素 — 每个元素都为信息服务 |
| **代表作品** | Müller-Brockmann 音乐会海报、瑞士国际主义风格 |
| **PPT 最佳用途** | 学术演示、建筑/设计评审、现代主义品牌幻灯片 |
| **色彩策略** | 有限调色板：主文字色 + 一个粗犷的几何强调色 + 白色/浅色背景 |
| **适配性** | ★★★★☆ — 非常适合结构化内容；不太适合情感/营销类幻灯片 |

### 04. Tufte — 数据真实

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 最大化数据墨水比。移除每一个非数据元素。迷你图、小多图和高密度显示。"最重要的是，展示数据。" |
| **SVG 转化** | 图表无网格线、无边框、无背景 — 只有数据标记和坐标轴标签；`fill-opacity` 和 `stroke-width` 取最小可用值；小多图布局在网格中重复相同尺寸；每页高密度数据点 |
| **代表作品** | Edward Tufte 的著作、迷你图、小多图网格 |
| **PPT 最佳用途** | 科学演示、研究报告、定量分析 |
| **色彩策略** | 墨水印纸：深色文字配白色/米色背景；颜色仅用于数据编码，绝不用于装饰 |
| **适配性** | ★★★☆☆ — 对数据优先的受众非常有力；对一般受众可能感觉过于简朴 |

## 流派二：动感诗学（动感·沉浸·技术美学）

### 05. Field.io — 生成式动态

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 让计算美学可视化。粒子系统、算法图案和数学之美。深色背景配发光数据点。技术即艺术。 |
| **SVG 转化** | 深色背景（`fill="#0A0A0A"`）；分散的 `<circle>` 元素配以不同的 `r` 和 `fill-opacity` 构建粒子场；节点间的细连接 `<line>` 元素（`stroke-opacity="0.15"`）；径向渐变聚光灯；强调色作为发光高亮 |
| **代表作品** | Field.io 数据雕塑、Refik Anadol 数据绘画 |
| **PPT 最佳用途** | AI/ML 产品发布、技术主题演讲、创新展示 |
| **色彩策略** | 深色画布 + 1-2 个发光强调色（电光蓝 #00D4FF、鲜绿 #00FF88 或暖琥珀 #FFB800） |
| **适配性** | ★★★☆☆ — 科技主题非常震撼；不适合企业/政府场景 |

### 06. Sagmeister — 表现性字体

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 字体排印即视觉艺术，不只是信息载体。字母变成插画。大小对比极端（8:1 或更大）。有趣但有目的地打破规则。 |
| **SVG 转化** | 英雄文字 120-180px `font-size`，创意定位（旋转、堆叠、重叠）；正文在强烈对比下（16-18px）；部分文字元素使用 `transform="rotate(...)"`；单词作为整页视觉宣言；每个"英雄时刻"限制 2-3 个词 |
| **代表作品** | Sagmeister & Walsh 品牌识别作品、实验性字体排印海报 |
| **PPT 最佳用途** | 创意机构提案、品牌识别演示、文化活动幻灯片 |
| **色彩策略** | 大胆出乎意料：珊瑚红 + 深黑、电光黄 + 炭灰，或单色配一个冲击性强调色 |
| **适配性** | ★★☆☆☆ — 冲击力很强但风险也大；需要自信的受众和清晰的品牌契合度 |

## 流派三：极简主义（秩序·留白·精致）

### 07. Build — 极端极简

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 减法是首要设计工具。如果去掉不影响意义，就必须去掉。视觉沉默和视觉元素同样重要。"更少，但更好。" |
| **SVG 转化** | 留白比例 ≥ 65%；每页最多 3 个视觉元素；无装饰图标、无边框、无背景图案；仅通过字号和字重建立文字层次（不用颜色区分）；单一强调色少量使用（每页 ≤ 1 个元素） |
| **代表作品** | Build 工作室品牌识别作品、Dieter Rams 产品设计哲学 |
| **PPT 最佳用途** | 主题演讲、高管总结、奢侈品牌幻灯片 |
| **色彩策略** | 近单色：米白（#FAFAFA）+ 炭灰（#1A1A1A）+ 一个柔和强调色 |
| **适配性** | ★★★★☆ — 低密度内容时非常优雅；不适合数据密集的页面 |

### 08. Notion — 功能性极简

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 简洁、系统化、工具般的清晰。通过一致的间距而非装饰建立信息层次。感觉像是一份恰好很美的井然有序的文档。 |
| **SVG 转化** | 所有间距基于一致的 8px 基线网格；浅色边框（`stroke="#E0E0E0" stroke-width="1"`）作为分隔线；正文用深灰色（#37352F）配白色背景；内联代码风格高亮（文字后加 `<rect rx="3" fill="#F7F6F3"/>`）；扁平图标 |
| **代表作品** | Notion 应用界面、Linear 应用设计、Stripe 文档 |
| **PPT 最佳用途** | 产品文档、工程评审、SaaS 产品幻灯片、内部工具演示 |
| **色彩策略** | 暖中性色：米白背景（#FFFFFF 或 #FAFAF8）+ 暖黑文字（#37352F）+ 微妙的蓝色或青色强调 |
| **适配性** | ★★★★☆ — 通用且安全；对外部营销可能略显低调 |

### 09. Apple — 电影感极简

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 产品为主角。深色背景上的戏剧性打光。巨大的字体配合极端的字重对比。电影级视觉叙事，每张幻灯片都是一个"场景"。 |
| **SVG 转化** | 深色背景（#000000 或 #1D1D1F）；产品图片覆盖 60%+ 页面面积；标题文字用 SF Pro 风格字体 56-72px，超细字重（300）或超粗（800）；文字用白色或近白色（#F5F5F7）；产品图片后方的戏剧性径向渐变聚光灯 |
| **代表作品** | Apple 发布会演示、Apple 产品页面 |
| **PPT 最佳用途** | 产品发布、硬件揭幕、高端品牌演示 |
| **色彩策略** | 黑 + 白 + 产品强调色（每个产品一种，如 iPhone 紫、MacBook 银） |
| **适配性** | ★★★☆☆ — 有产品图片时非常震撼；没有高质量照片则效果大打折扣 |

## 流派四：实验先锋（先锋·生成·冲击）

### 10. Instrumental — 几何精准

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 几何形状既是结构系统也是装饰系统。圆形、三角形和线条同时构成网格和装饰。元素间的数学关系。 |
| **SVG 转化** | 使用低透明度的 `<circle>`、`<line>` 和 `<polygon>` 构建背景几何图案（`fill-opacity="0.05"`）；内容卡片使用几何裁剪形状；强调装饰为精确的几何构造；主要布局分割使用黄金比例（1.618） |
| **代表作品** | Instrumental 设计工作室、几何品牌识别系统 |
| **PPT 最佳用途** | 建筑演示、工程展示、数学/科学主题 |
| **色彩策略** | 建筑感：混凝土灰 + 蓝图蓝 + 铜/黄铜强调色 |
| **适配性** | ★★★☆☆ — 结构化/技术内容很有力；对叙事类内容过于刚硬 |

### 11. Atelier Olschinsky — 数字拼贴

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 层叠的视觉复杂性。多个重叠的信息平面。半透明元素创造纵深。通过元素堆积而非表面处理获得丰富质感。 |
| **SVG 转化** | 多个 `<rect>` 元素使用 `fill-opacity="0.05-0.15"` 创建纵深层次；不同大小的重叠文本块；内容块刻意延伸到安全边距外（由 viewBox 裁剪）；渐变叠加创造氛围纵深 |
| **代表作品** | Atelier Olschinsky 海报、层叠编辑设计 |
| **PPT 最佳用途** | 创意行业演示、艺术/文化活动、品牌探索幻灯片 |
| **色彩策略** | 丰富且有层次：3-4 种半透明颜色通过重叠创造纵深（如深海军蓝 + 珊瑚色 + 金色，不同透明度） |
| **适配性** | ★★☆☆☆ — 视觉非常丰富但可能过度刺激；最适合设计素养高的受众 |

## 流派五：东方哲学（温润·诗意·思辨）

### 12. Takram — 东方设计思维

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 技术与人文的交汇。温暖的精确。看起来像手绘但数学上精确的图解。元素间的空白承载意义（間 — Ma）。 |
| **SVG 转化** | 充裕的留白（70%+ 页面面积）；细描边连接概念的线条（`stroke-width="1"`）；温暖的背景色调（#FAFAF5 奶油色或 #F5F0EB 暖灰）；用简单 SVG 形状绘制的小巧精确插图；文字以有意的不对称方式定位 |
| **代表作品** | Takram 设计项目、日本编辑设计 |
| **PPT 最佳用途** | 设计思维工作坊、创新战略、以人为本的设计报告、跨文化演示 |
| **色彩策略** | 暖中性色：奶油色 + 暖灰 + 一个自然灵感强调色（苔藓绿 #5B7B4F、靛蓝 #3D4F7C 或赤陶色 #C4785B） |
| **适配性** | ★★★★☆ — 普遍优雅；对思辨性/反思性内容尤其有力 |

### 13. Kenya Hara — 无印良品哲学（虚空即充盈）

| 属性 | 描述 |
|------|------|
| **Visual DNA** | "空并非无物 — 而是承载可能性的容器。"白色不是空白，而是蓄势。字体排印克制到极致成为雕塑。通过"不展示什么"来传达。 |
| **SVG 转化** | 背景：纯白（#FFFFFF）；文字：暖深灰（#333333）而非纯黑；每页最多 15-20 个词；巨大的边距（120px+ 四边）；每页最多一个视觉元素；仅通过字重差异建立层次（300 vs 700） |
| **代表作品** | 无印良品品牌设计、Kenya Hara 的展览设计 |
| **PPT 最佳用途** | 哲学/思想领导力、奢侈品牌叙事、年度回顾、沉思型演示 |
| **色彩策略** | 白 + 暖灰 + 至多一个柔和的大地色调。颜色是事件，不是常态。 |
| **适配性** | ★★★☆☆ — 极为优雅但要求非常低的内容密度；不适合数据密集的幻灯片 |

### 14. Naoto Fukasawa — Without Thought（无意识设计）

| 属性 | 描述 |
|------|------|
| **Visual DNA** | 自然到不被注意的设计。观众关注内容而非设计。没有任何元素在吸引注意力。不可见的匠心让一切感觉毫不费力且理所当然。 |
| **SVG 转化** | 系统默认感的字体排印（干净的无衬线字体、标准字重）；一致的间距不引起注意；完全没有装饰元素；仅通过位置和大小建立信息层次；每个设计决策服务于功能，没有什么仅服务于美学 |
| **代表作品** | Naoto Fukasawa 产品设计、±0 品牌设计 |
| **PPT 最佳用途** | 内部团队更新、项目评审、运营报告 — 内容本身就是演示的场景 |
| **色彩策略** | 隐形调色板：浅灰背景（#F5F5F5）+ 深色文字（#2D2D2D）+ 功能性强调色（蓝色表示链接、绿色表示正面、红色表示警报） |
| **适配性** | ★★★★★ — "无设计的设计" — 适用于一切，通过克制打动懂设计的观众 |

## 如何将哲学注入 design_spec.md

用户选定哲学后：

1. **Mood Statement**（第 V 节）：写 1-2 句话捕捉该哲学的精髓
   - 示例（Pentagram）："一份如 Pentagram 品牌识别系统般的高管战略幻灯片 — 极致的字体层次、瑞士网格精度和 60% 留白。"
2. **Color Scheme**（第 III 节）：遵循该哲学的色彩策略
3. **Typography**（第 IV 节）：调整字号比例以匹配该哲学的层次风格
4. **Layout**（第 V 节）：应用该哲学的间距和构图原则
5. **Icon Usage**（第 VI 节）：匹配该哲学的视觉密度预期（极简哲学 → 更少图标；信息密集哲学 → 仅功能性图标）
