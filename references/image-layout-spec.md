# 图片布局规范（强制执行）

包含图片的页面的布局规则。策略师规划阶段和执行器生成阶段都必须遵循这些规则。

**核心原则：根据图片原始宽高比计算布局，确保图片完整显示，不产生多余留白或裁切。**

---

## 布局决策流程

```
1. 获取图片原始尺寸 → 计算宽高比（宽/高）
2. 根据宽高比选择布局类型
3. 计算图片的最大显示尺寸
4. 分配剩余空间给文本区域
5. 将结果填入设计规范的图片资源列表
```

**执行时机**：如果图片方案包含"B) 用户提供"，在策略师完成八大确认之后、内容分析和大纲编写之前，必须运行扫描并填充图片资源列表。

---

## 布局类型选择（强制执行）

| 图片宽高比 | 布局类型 | 图片位置 | 说明 |
|------------|----------|----------|------|
| > 2.0（超宽） | 上下分割 | 顶部全宽 | 图片跨满画布宽度，高度按比例 |
| 1.5-2.0（宽幅） | 上下分割 | 顶部 | 图片宽 = 内容区域宽，高度按比例 |
| 1.2-1.5（标准） | 左右分割 | 左侧 | 图片以高度优先适配，宽度按比例 |
| 0.8-1.2（方形） | 左右分割 | 左侧 | 图片占满内容区域高度，宽度按比例 |
| < 0.8（竖版） | 左右分割 | 左侧 | 图片高 = 内容区域高，宽度按比例 |

> 边界情况：当宽高比处于临界值（如 1.5）时，根据文本量决定。文本多 → 左右分割；文本少 → 上下分割。

## 尺寸计算公式

### 画布参数（所有格式）

| 格式 | 画布 | 边距（左右, 上下） | 内容区域（宽 x 高） | 标题高度 | 内容起始 Y |
|------|------|-------------------|---------------------|----------|-----------|
| PPT 16:9 | 1280x720 | 60, 60 | 1160 x 600 | 60px | 80px |
| PPT 4:3 | 1024x768 | 50, 50 | 924 x 608 | 60px | 70px |
| 小红书 | 1242x1660 | 60, 80 | 1122 x 1500 | 80px | 100px |
| 微信朋友圈 | 1080x1080 | 60, 60 | 960 x 960 | 60px | 80px |
| Story | 1080x1920 | 60, 120/180 | 960 x 1620 | 80px | 140px |
| 微信文章 | 900x383 | 40, 40 | 820 x 303 | 40px | 50px |

> 以下公式中，**W** = 内容区域宽，**H** = 内容区域高（已扣除标题）。PPT 16:9 示例使用 W=1160, H=600。

### 上下布局计算

```
图片宽 = W = 1160 px
图片高 = W / R = 1160 / R px
文本区域高 = H - 图片高 - 间距(20px)

验证：文本区域高 >= 150px（至少容纳 3-4 行文本）
如不满足 → 切换为左右布局
```

### 左右布局计算

**方法 1（高度优先，适合竖版图片）**：
```
图片高 = H = 600 px
图片宽 = H x R = 600 x R px
文本区域宽 = W - 图片宽 - 间距(20px)
```

**方法 2（宽度约束，适合宽幅图片转为左右布局）**：
```
图片宽 = W x 0.7 = 812 px
图片高 = 图片宽 / R
文本区域宽 = W - 图片宽 - 间距(20px)
```

**验证**：文本区域宽 >= 280px；否则缩小图片区域宽度。

## 布局示例

### 超宽图片（宽高比 2.45）

```
原始：1960x800, R=2.45 → 上下分割
图片：1160x473，文本区域：1160x147 → 7:3 上下分割
```

### 标准横幅（宽高比 1.38）

```
原始：1614x1171, R=1.38 → 左右分割
图片：773x560（左侧），文本区域：367x560（右侧）→ 7:3 左右分割
```

### 宽幅图片边界情况（宽高比 1.75）

```
原始：1820x1040, R=1.75
尝试上下分割：图片高=663，文本区域=-43 ❌
切换为左右分割：图片 780x446（左侧），文本区域 360x600（右侧）→ 7:3 左右分割
```

## 竖版画布覆盖规则

默认的布局类型选择表假设**横版或方形画布**。对于竖版画布（高 > 宽），规则有所不同，因为左右分割会使两列都过窄。

| 画布方向 | 图片宽高比 | 推荐布局 | 原因 |
|----------|-----------|----------|------|
| 竖版（小红书, Story） | > 1.5（宽幅） | 上下分割 | 与横版画布一致 |
| 竖版（小红书, Story） | 1.2-1.5（标准） | 上下分割 | 竖版画布上左右分割太窄 |
| 竖版（小红书, Story） | 0.8-1.2（方形） | 上下分割 | 图片适合放在上半部分 |
| 竖版（小红书, Story） | 0.5-0.8（竖版） | 左右分割 | 竖版图片在竖版画布上可行 |
| 竖版（小红书, Story） | < 0.5（极端竖版） | 左右分割 | 图片占一侧，文本占另一侧 |

> 方形画布（微信朋友圈 1:1）使用标准横版画布规则。

## 多图布局

当一张幻灯片包含多张图片时，使用以下公式均匀划分内容区域。

### 网格公式

```
columns = 列数
rows = 行数
gap = 20px（PPT 格式）或 30px（社交格式）

cell_width  = (W - (columns - 1) * gap) / columns
cell_height = (H - (rows - 1) * gap) / rows
```

### 常见模式

| 图片数量 | 布局 | 网格 | 说明 |
|----------|------|------|------|
| 2（均为横版） | 并排 | 2x1 | 两列等宽 |
| 2（均为竖版） | 上下堆叠 | 1x2 | 两行等高 |
| 2（混合） | 1 大 + 1 小 | 自定义 | 横版在上（全宽），竖版在右下 |
| 3 | 1 大 + 2 小 | 1+2 | 左侧大图（50% 宽），右侧两张堆叠 |
| 4 | 网格 | 2x2 | 等大单元格 |

### 示例：PPT 16:9 上的 2x2 网格

```
W=1160, H=600, gap=20
cell_width  = (1160 - 20) / 2 = 570
cell_height = (600 - 20) / 2 = 290

图片位置：
  (60, 80)   570x290    (650, 80)  570x290
  (60, 390)  570x290    (650, 390) 570x290
```

> 多图幻灯片中，所有图片使用 `preserveAspectRatio="xMidYMid meet"` 以保持单元格内显示的一致性。

## 禁止做法

| 禁止行为 | 正确做法 |
|----------|----------|
| 固定 50:50 或随意比例 | 根据图片宽高比动态计算 |
| 将宽幅图片强塞进方形容器 | 使用上下布局或增大图片区域宽度 |
| 将竖版图片放在狭窄的横向条带中 | 使用左右布局，图片在左 |
| 图片留白超过 10% | 重新计算布局或选择替代方案 |
| 裁切图片关键内容 | 使用 `preserveAspectRatio="xMidYMid meet"` |
| 文本区域小到无法阅读 | 确保文本区域 >= 150px（上下布局）或 >= 280px（左右布局） |

## SVG 图片嵌入代码

### 完整显示（推荐用于数据图表）

```xml
<image href="../images/xxx.png"
       x="60" y="80" width="780" height="446"
       preserveAspectRatio="xMidYMid meet"/>
```

### 裁切填充（仅用于背景）

```xml
<image href="../images/bg.png"
       x="0" y="0" width="1280" height="720"
       preserveAspectRatio="xMidYMid slice"/>
```

## 图片资源列表模板

在设计规范与内容大纲中，图片资源列表必须包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| Filename | 图片文件名 | `p12_0.png` |
| Original dimensions | 原始尺寸（宽 x 高） | 1524x968 |
| Ratio | 宽高比（宽 / 高） | 1.57 |
| Page | 使用页码 | 第 5 页 |
| Type | 视觉类型 | 背景 / 摄影 / 插画 / 图表 / 装饰 |
| Layout plan | 上下/左右 + 分割比例 | 上下 6:4 或 左右 7:3 |
| Image area | 图片显示尺寸 | 1160x420 或 780x446 |
| Text area | 剩余空间尺寸 | 1160x200 或 360x600 |

**Type 字段供 Image_Generator 选择合适的提示词策略使用。**

## 自动化工具

```bash
python3 scripts/analyze_images.py <project_path>/images                    # 默认：PPT 16:9
python3 scripts/analyze_images.py <project_path>/images --canvas ppt43     # PPT 4:3
python3 scripts/analyze_images.py <project_path>/images --canvas xiaohongshu  # 小红书
```

`--canvas` 参数选择目标画布格式（默认 `ppt169`）。工具根据本规范中的公式计算布局类型（上下 / 左右）、图片显示区域和文本区域尺寸。输出包含 Markdown 表格，可直接粘贴到图片资源列表中。
