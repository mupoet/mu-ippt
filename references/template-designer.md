# Template Designer — 模板设计角色

## 核心使命

基于最终确认的模板简报，为**全局模板库**生成可复用的页面模板。

> 这是一个独立角色：仅通过 `/create-template` 工作流触发。它**不是**主 PPT 生成流水线中的项目级模板选择/定制步骤。

## 使用方式

- **触发方式**：`/create-template` 工作流
- **输出位置**：`templates/layouts/<template_name>/`
- **输入**：最终确认的模板简报（模板 ID、显示名称、分类、适用场景、风格基调、主题模式、画布格式、可选参考素材）

当工作流提供 PPTX 参考源文件时，实际输入包来自统一的 `pptx_template_import.py` 准备工作区，包含：

- 最终确认的模板简报
- `manifest.json`
- `analysis.md`
- `normalized_assets.json`
- 导出的 `assets/`
- 清理后的幻灯片 SVG 参考（来自 `svg/`）
- `reference_svg_selection.json`
- 可选的截图（用于视觉交叉校验）

PPTX 驱动的模板创建输入优先级：

1. `manifest.json` 用于事实性元数据
2. `normalized_assets.json` 用于规范化素材决策
3. 导出的 `assets/` 用于可复用的视觉资源
4. `analysis.md` 用于页面类型指导
5. `reference_svg_selection.json` 用于决定优先检查哪些导出的 SVG 页面
6. 清理后的幻灯片 SVG 参考用于检查构图、间距和固定装饰线索
7. 截图 / 原始 PPTX 仅用于风格验证

## 核心模板清单

| # | 文件名 | 用途 | 说明 |
|---|--------|------|------|
| 01 | `01_cover.svg` | 封面 | 固定结构：标题、副标题、日期、组织名称 |
| 02 | `02_chapter.svg` | 章节页 | 固定结构：章节编号、章节标题 |
| 03 | `03_content.svg` | 内容页 | 灵活结构：仅定义页头/页脚；内容区域由 AI 自由排版 |
| 04 | `04_ending.svg` | 结尾页 | 固定结构：致谢语、联系方式 |
| -- | `02_toc.svg` | 目录页 | 可选：目录标题、章节列表（编号 + 标题） |

**设计理念**：模板定义视觉一致性和结构性页面；内容页保持最大灵活性。

**命名说明**：目录页保持 `02_toc.svg` 命名，以兼容模板库排序。

### 可选扩展页面（按需创建）

- 过渡 / 子章节页（如 `05_section_break.svg`）
- 附录页（如 `06_appendix.svg`）
- 免责声明 / 保密页（如 `07_disclaimer.svg`）

## 模板设计规范

### 1. 必须生成 design_spec.md

创建全局模板时，必须生成 `design_spec.md`，包含：

```markdown
# [模板名称] - 设计规范

## I. 模板概述（名称、使用场景、设计基调）
## II. 画布规格（16:9、1280x720、viewBox）
## III. 配色方案（主色、辅色、强调色 HEX 值）
## IV. 字体体系（字体栈、字号层级）
## V. 页面结构（通用布局、装饰设计）
## VI. 页面类型（4 种核心页面类型）
## VII. 布局模式（推荐方案）
## VIII. 间距规范
## IX. SVG 技术约束
## X. 占位符规范
```

### 2. 继承设计规范

模板必须严格遵循最终确认的模板简报和生成的 `design_spec.md`：
- **画布尺寸**：viewBox 与设计规范一致
- **配色方案**：使用规范中的主色、辅色和强调色
- **字体方案**：使用规范中的字体预设
- **布局原则**：边距和间距符合规范

如果存在 PPTX 导入输出：
- 优先使用导入的主题色和字体，而非视觉猜测值
- 在 `normalized_assets.json` 中全局有意义的情况下，复用规范化的背景和 Logo
- 将 `analysis.md` 中的页面类型候选项视为参考提示，而非确定结论

**前置条件**：

- 如果提供了 `reference_svg_selection.json`，在读取所有选定的参考 SVG 文件之前，不要生成任何模板 SVG 或 `design_spec.md`
- 在开始生成模板之前，明确报告已读取的幻灯片索引

### 2.1 PPTX 导入简化规则

导入的 PPTX 是**参考来源**，不是直接转换目标。

应该做的：
- 保留品牌素材、重复出现的背景和稳定的结构母题
- 将布局重构为符合 PPT Master 约束的干净 SVG 结构
- 将重复的装饰片段简化为数量更少、更易维护的 SVG 元素
- 当原始装饰层过于复杂难以干净重建时，使用背景图片素材
- 当 AI 规范化判断它们代表同一可见图像时，优先使用原始导出素材（如 `image1.png`）而非匹配的 `inline_*` 素材
- 将 `inline_*` 素材主要视为导出参考层，而非最终模板素材名
- 使用清理后的幻灯片 SVG 参考来检查构图、间距、文本层级和固定装饰结构，但仅在事实性元数据已锚定之后
- 如果需要检查的清理后 SVG 参考页面 `<= 10` 页，全部读取；如果超过 `10` 页，只读取 `10` 个代表性页面

不应该做的：
- 尝试 1:1 翻译每个 PowerPoint 形状、分组、阴影或装饰片段
- 当 PPT 特有的复杂性会使 SVG 变脆弱或难以编辑时，镜像这种复杂性
- 引入密集的低价值矢量细节，而这些细节并不实质性提升模板的可复用性
- 将仅作为遮罩/透明辅助层的 `inline_*` 素材提升为最终模板包的一部分，除非它们是唯一可行的可复用表示

### 2.2 原始素材与 Inline 素材的规范化

当 `normalized_assets.json` 存在时，遵循以下规则：

1. 如果原始导出素材和 `inline_*` 素材明确对应同一可见图像，使用**原始导出素材**作为规范模板源。
2. 如果 `inline_*` 素材没有可靠的原始导出对应物，可以保留为派生模板素材。
3. 如果 `inline_*` 素材仅作为 SVG 导出输出中的遮罩、透明辅助层或其他非语义支持层，默认不将其纳入最终模板素材集。
4. 最终模板素材名应具有语义性，如 `cover_bg.png` 或 `brand_emblem.png`，而不是 `image3.png` 或 `inline_abcd1234.png`。

### 3. 占位符标记

为可替换内容使用清晰的占位符标记：

```xml
<!-- 文本占位符 -->
<text x="80" y="320" fill="#FFFFFF" font-size="48" font-weight="bold">
  {{TITLE}}
</text>

<!-- 内容区域占位符（仅内容页） -->
<rect x="40" y="90" width="1200" height="550" fill="#FFFFFF" rx="8"/>
<text x="640" y="365" text-anchor="middle" fill="#CBD5E1" font-size="16">
  {{CONTENT_AREA}}
</text>
```

### 4. 占位符参考表

| 占位符 | 用途 | 适用模板 |
|--------|------|----------|
| `{{TITLE}}` | 主标题 | 封面 |
| `{{SUBTITLE}}` | 副标题 | 封面 |
| `{{DATE}}` | 日期 | 封面 |
| `{{AUTHOR}}` | 作者 / 组织 | 封面 |
| `{{CHAPTER_NUM}}` | 章节编号 | 章节页 |
| `{{CHAPTER_TITLE}}` | 章节标题 | 章节页 |
| `{{CHAPTER_DESC}}` | 章节描述 | 章节页 |
| `{{PAGE_TITLE}}` | 页面标题 | 内容页 |
| `{{KEY_MESSAGE}}` | 核心要点 | 内容页（咨询风格） |
| `{{CONTENT_AREA}}` | 内容区域 | 内容页 |
| `{{SECTION_NAME}}` | 章节名称 | 内容页页脚 |
| `{{SOURCE}}` | 数据来源 | 内容页页脚 |
| `{{PAGE_NUM}}` | 页码 | 内容页、结尾页 |
| `{{THANK_YOU}}` | 致谢语 | 结尾页 |
| `{{ENDING_SUBTITLE}}` | 结尾副标题 | 结尾页 |
| `{{CLOSING_MESSAGE}}` | 结束语 | 结尾页 |
| `{{CONTACT_INFO}}` | 联系方式 | 结尾页 |
| `{{COPYRIGHT}}` | 版权信息 | 结尾页 |

对于**新建库模板**中的目录页，使用索引式占位符：

- `{{TOC_ITEM_1_TITLE}}`、`{{TOC_ITEM_1_DESC}}`
- `{{TOC_ITEM_2_TITLE}}`、`{{TOC_ITEM_2_DESC}}`
- ...

**不要**为新模板创建新的目录占位符族（如 `{{CHAPTER_01_TITLE}}`）。已有模板可能包含历史遗留的占位符变体，但新建库资产应统一使用索引式目录约定。

从导入的 PPTX 参考重建时，占位符插入优先于视觉模仿。如果原始布局没有为规范占位符留出足够空间，应调整布局而不是发明一次性的占位符族。

## 输出要求

### 文件保存位置

```
templates/layouts/<template_name>/
├── design_spec.md     # 设计规范（必需）
├── 01_cover.svg
├── 02_chapter.svg
├── 02_toc.svg          # 可选
├── 03_content.svg
├── 04_ending.svg
└── *.png / *.jpg       # 图片素材（如有）
```

### 模板预览

每个模板生成后，提供简要的汇总表格，列出每个模板的状态。

如果模板基于 PPTX 导入输出，简要说明：
- 哪些提取的素材被直接复用
- 哪些复杂的原始装饰被有意简化
- 是否有页面类型映射需要超出导入启发式规则的人工判断

## 使用预建模板库（可选）

如果已有合适的模板资源，直接使用而无需重新生成：

1. **复制模板**：将模板文件复制到项目的 `templates/` 目录
2. **调整颜色**：根据项目设计规范修改颜色
3. **定制化**：进行项目特定的调整

本节描述的是下游复用。`Template_Designer` 角色本身负责首先创建或规范化可复用的库资产。

**示例库结构**（查询 `templates/layouts/layouts_index.json`）：

```
templates/layouts/
├── exhibit/           # Exhibit 风格（结论先行，数据驱动）
├── 科技蓝商务/         # 科技蓝商务风格
└── smart_red/         # 智慧红橙风格
```
