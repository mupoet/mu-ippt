# SVG 技术图表布局最佳实践（Layout Best Practices）

本规范定义 SVG 技术图表的布局规则，确保组件间距、箭头路由、标签放置和碰撞检测的一致性。

**画布基准**：PPT 大师默认画布为 `1280×720`（viewBox="0 0 1280 720"），所有坐标和间距规则基于此尺寸。

---

## 1. 组件间距规则

| 规则 | 最小值 | 推荐值 | 说明 |
|------|--------|--------|------|
| 同层组件水平间距 | 80px | 100–120px | 组件边缘到边缘 |
| 层间垂直间距 | 100px | 120px | 上层底边到下层顶边 |
| 箭头路径净空 | 60px | 80px | 箭头路径到最近组件边缘 |
| 画布边距 | 40px | 60px | 内容到画布边缘 |
| 组件内边距 | 8px | 12px | 文字到组件边框 |

### 网格对齐

- 坐标对齐到 **8px 网格**（x/y 值为 8 的倍数）
- 水平节点位置间隔推荐 **120px**
- 垂直节点位置间隔推荐 **120px**

---

## 2. 箭头路由规则

### 连接点规范

- **禁止**将箭头连接到组件的角落——使用边缘中点
- 连接点到角落的最小距离：**20px**
- 同一边有多个箭头时，错开间距 **±30px**

```xml
<!-- ✅ 正确：箭头连接到右侧边缘中点 -->
<line x1="260" y1="200" x2="400" y2="200"
      stroke="#2563eb" stroke-width="2" marker-end="url(#arrow-primary)"/>

<!-- ❌ 错误：箭头连接到角落 -->
<line x1="260" y1="160" x2="400" y2="240"
      stroke="#2563eb" stroke-width="2" marker-end="url(#arrow-primary)"/>
```

### 正交路由（Orthogonal Routing）

优先使用 L 形或 Z 形路径，避免对角线穿越组件：

```xml
<!-- ✅ 正确：正交路由绕过组件 -->
<path d="M 200,100 L 200,250 L 600,250 L 600,400"
      fill="none" stroke="#2563eb" stroke-width="2"
      marker-end="url(#arrow-primary)"/>

<!-- ❌ 错误：对角线穿越组件 -->
<line x1="200" y1="100" x2="600" y2="400"
      stroke="#2563eb" stroke-width="2"/>
```

### 曲线路径

控制点到组件边缘的安全距离：**≥40px**

```xml
<!-- 安全的曲线路由 -->
<path d="M 200,100 Q 400,200 600,400"
      fill="none" stroke="#7c3aed" stroke-width="1.5"
      marker-end="url(#arrow-feedback)"/>
<!-- 控制点 (400,200) 距所有组件 ≥40px -->
```

### 并行箭头

同层间有多条箭头时，Y 坐标错开 **15–20px**，防止视觉重叠。

## 3. Jump-over 弧线规范（交叉处理）

当两条箭头路径不可避免地交叉时，**必须**使用 jump-over 弧线：

### 规则

- 弧线半径：**5px**（标准），多重交叉递增为 **7px、9px**
- 由"下层"箭头绘制弧线跳过"上层"箭头
- 弧线 `fill="none"`，`stroke` 与箭头颜色一致

### SVG 实现

```xml
<!-- 水平箭头在交叉点处跳过垂直箭头 -->
<!-- 假设交叉点 x=400 -->
<path d="M 200,300 L 395,300 A 5,5 0 0,1 405,300 L 600,300"
      fill="none" stroke="#2563eb" stroke-width="2"
      marker-end="url(#arrow-primary)"/>
```

**A 命令说明**：`A rx,ry rotation large-arc-flag sweep-flag x,y`
- `A 5,5 0 0,1 405,300` = 半径 5px，顺时针（sweep=1），跳到 (405,300)

### 多重交叉

```xml
<!-- 三条交叉时，递增弧线半径 -->
<!-- 交叉 1 (x=300): r=5 -->
<path d="M 100,300 L 295,300 A 5,5 0 0,1 305,300 L ..."
<!-- 交叉 2 (x=500): r=7 -->
      ... L 493,300 A 7,7 0 0,1 507,300 L 700,300"/>
```

## 4. 箭头标签规范

### 必须有背景矩形

箭头上的文字标签**必须**有半透明背景矩形，防止与线条重叠：

```xml
<!-- 标签背景 + 文字 -->
<rect x="296" y="188" width="48" height="18" rx="3"
      fill="#ffffff" fill-opacity="0.95"/>
<text x="320" y="201" text-anchor="middle"
      font-size="11" fill="#475569"
      font-family="Microsoft YaHei, Arial, sans-serif">query</text>
```

### 标签放置规则

| 规则 | 值 |
|------|-----|
| 位置 | 箭头路径中点，垂直偏移 5–10px |
| 安全距离 | 距组件边缘 ≥15px |
| 最大字数 | ≤3 个词 |
| 背景内边距 | 水平 4px，垂直 2px |
| 背景透明度 | 0.90–0.95 |
| 多标签错开 | 垂直方向 20px |

## 5. Z-Index 分层顺序

SVG 按文档顺序渲染（后绘制的在上层），严格按以下顺序排列：

```
层 1（最底层）：背景矩形
层 2：分组容器（虚线矩形）
层 3：箭头路径
层 4：箭头标签背景矩形
层 5：组件（矩形、圆柱、六边形等）
层 6：组件文字
层 7：箭头标签文字
层 8（最顶层）：图例（Legend）
```

### 半透明填充的箭头遮罩

当组件使用半透明填充（如 `fill-opacity="0.4"` 或 `rgba(..., 0.4)`）时，层3的箭头会透过层5的组件显示出来。解决方法：在半透明组件下方垫一层**不透明背景rect**，遮住穿过的箭头。

```xml
<!-- ✅ 正确：双层 rect 遮罩箭头 -->
<!-- 层 4.5：不透明底层，遮住下方箭头 -->
<rect x="400" y="200" width="160" height="60" rx="6"
      fill="#ffffff"/>
<!-- 层 5：半透明样式层 -->
<rect x="400" y="200" width="160" height="60" rx="6"
      fill="#FFD100" fill-opacity="0.15"
      stroke="#FFD100" stroke-width="1.5"/>

<!-- ❌ 错误：单层半透明，箭头透出 -->
<rect x="400" y="200" width="160" height="60" rx="6"
      fill="#FFD100" fill-opacity="0.15"
      stroke="#FFD100" stroke-width="1.5"/>
```

> **规则**：只要组件填充不是完全不透明（opacity < 1），就必须使用双层 rect。底层 fill 取画布背景色（白底用 `#ffffff`，暗色用 `#1A1A2E`）。

## 6. 碰撞检测检查表

在最终输出 SVG 前，逐项检查：

- [ ] **箭头-组件碰撞**：所有箭头路径不穿过组件内部（用正交路由绕行）
- [ ] **文字溢出**：所有文字在组件内留有 ≥8px 内边距（估算：`text.length × 7px ≤ shape_width - 16px`）
- [ ] **标签-组件重叠**：箭头标签距组件边缘 ≥15px
- [ ] **组件重叠**：所有组件边界框无交叉（保持 ≥10px 安全间距）
- [ ] **箭头交叉**：交叉处使用 jump-over 弧线
- [ ] **箭头标签背景**：每个箭头标签都有 `<rect>` 背景
- [ ] **连接点合规**：箭头端点在组件边缘中点附近（距角落 ≥20px）
- [ ] **图例可读**：图例不与内容重叠
- [ ] **标签无遮挡**：多个标签间无重叠

## 7. 容器规范

### 进出规则

箭头进出容器（Swim Lane）时，优先从**组件间的空隙**进入，不要穿过内部组件的身体。

### 容器嵌套

- 最多嵌套 **2 层**容器
- 嵌套容器间距 ≥**20px**
- 内层容器使用更细的描边（0.8px vs 外层 1px）

## 8. Anti-AI 视觉审查（每页 SVG 输出前必过）

命中任何一条必须修正后再输出：

**布局：**
- [ ] **禁止等宽三列** — 三个等宽并排面板是 AI 生成的头号特征。用 `2:1`、`3:2` 等非对称比例
- [ ] **禁止默认居中标题** — 标题左对齐或非对称布局优先；居中只在刻意强调时使用
- [ ] **至少一个差异化面板** — 同页多个面板时，至少有一个在尺寸、底色或排版上与其他不同

**配色：**
- [ ] **禁止纯黑 `#000000`** — 正文用 `#1a1a1a`、`#2d2a26` 或风格定义的深色替代
- [ ] **强调色 ≤ 1 个，饱和度 < 80%** — 不要多彩轰炸；多色需求通过明度/灰度变化区分
- [ ] **禁止霓虹渐变** — 不要紫蓝 AI 光效、渐变填充标题

**排版：**
- [ ] **层级靠字重+色彩，不只靠字号** — 不要仅用 font-size 递增区分层级；搭配 font-weight 和颜色深浅
- [ ] **色温一致** — 同一张图内不要冷暖色混搭（暖底+冷强调 或反之）

## 9. SVG 正交连线规范

所有箭头/连线必须遵循正交路由（水平+垂直线段），使用 `<path>` 的 `M`/`L` 命令：

```xml
<!-- ✅ 正交连线（垂直→水平→垂直） -->
<defs>
  <marker id="arrowhead" markerWidth="8" markerHeight="6" refX="8" refY="3" orient="auto">
    <path d="M0,0 L8,3 L0,6" fill="none" stroke="{{ARROW_COLOR}}" stroke-width="1"/>
  </marker>
</defs>
<path d="M 200,72 L 200,90 L 400,90 L 400,108"
      fill="none" stroke="{{ARROW_COLOR}}" stroke-width="1.5"
      marker-end="url(#arrowhead)"/>

<!-- ❌ 禁止对角线直连 -->
<line x1="200" y1="72" x2="400" y2="108" .../>
```

**连线规则：**
- 严格使用 `M`/`L` 命令画正交线段，禁止 Bézier 曲线（`C`/`Q`）和对角线（`<line>` 斜向）
- 虚线用 `stroke-dasharray="6 4"` class 区分
- 同一图表内 marker 定义复用（`<defs>` 统一定义 arrowhead）
- 同一边多条连线错开 ±30px 避免重叠

## 常见反模式与修正

| 反模式 | 修正方案 |
|--------|----------|
| 箭头穿越组件 | 使用正交路由绕行 |
| 曲线控制点距组件太近 | 控制点距组件 ≥40px |
| 标签无背景导致文字难以辨认 | 添加 `<rect>` 背景 |
| 组件间距 < 80px | 增加间距或缩小组件 |
| 箭头连接到角落 | 移至边缘中点偏移位置 |
| 无 Z-Index 规划 | 按分层顺序组织 SVG 元素 |
| 箭头交叉无 jump-over | 添加 5px 弧线跳过 |
