# 品牌资产协议

> 当演示文稿涉及特定品牌或公司时，在进入八项确认**之前必须**执行本协议。目标：收集真实的品牌资产，让 PPT 看起来像品牌自己的作品，而不是随便贴个 logo 的通用模板。

**核心理念：资产 > 规范**

一个真实的 logo 比一套完美的配色方案更有价值。一张产品截图传递的品牌感比一个 HEX 色值更强。始终优先使用具体的视觉资产，而非抽象的描述。

---

## 品牌资产优先级排序

| 优先级 | 资产类型 | 品牌识别价值 | 备注 |
|--------|---------|------------|------|
| 1（最高） | **Logo**（SVG/PNG，浅色和深色版本） | ★★★★★ | 最具辨识度的品牌元素 |
| 2 | **产品图片 / 渲染图** | ★★★★☆ | 真实产品照片 > 任何插画 |
| 3 | **UI 截图** | ★★★☆☆ | 展示品牌的实际视觉语言 |
| 4 | **品牌色彩**（已验证的 HEX 值） | ★★★☆☆ | 必须从真实资产中提取，不能猜测 |
| 5 | **品牌字体** | ★★☆☆☆ | PPT 中通常使用系统安全字体替代即可 |
| 6（最低） | **品牌情绪关键词** | ★☆☆☆☆ | "高端、创新、可信赖"——单独使用太模糊 |

---

## 5 步硬流程

### 步骤 1：询问用户

在进行任何搜索之前，先问用户有哪些可用资产。使用以下清单：

```
📋 品牌资产清单 — 请提供你手上有的材料：
□ Logo 文件（优选 SVG，PNG 也可；如有浅色+深色版本请都提供）
□ 品牌指南 / 视觉规范（PDF 或链接）
□ 产品照片或渲染图
□ UI 截图或应用商店截图
□ 官方色值（HEX 值）
□ 品牌字体（或字体名称）
□ 现有的演示文稿模板
□ 喜欢的参考材料（竞品 PPT、网站等）
```

**如果用户提供了资产**：直接跳到步骤 4（验证 + 提取）。
**如果用户什么都没有**：进入步骤 2。

### 步骤 2：搜索官方渠道

系统性地搜索品牌资产。按资产类型的搜索策略：

| 资产类型 | 搜索路径 | 备选方案 |
|---------|---------|---------|
| **Logo** | 官网页脚/页头 → 新闻/媒体资源页 → 品牌指南 PDF | 社交媒体头像（低分辨率，最后手段） |
| **产品图片** | 官方产品页 → 新闻稿 → 电商平台列表 | 含产品照片的公关新闻报道 |
| **品牌色彩** | 品牌指南 PDF → 从 logo/网站提取 → 开发者工具检查 | 从产品摄影中提取主色调 |
| **品牌字体** | 品牌指南 → 网站 `font-family` 检查 | 映射到最接近的系统安全字体 |

**搜索纪律**：
- 优先使用官方渠道（公司网站、新闻资源包、投资者关系页面）
- 色值至少交叉验证 2 个来源
- 未经用户明确同意，不使用粉丝制作或非官方资产

### 步骤 3：下载资产

对每个发现的资产，按相应路径下载：

| 资产 | 首选路径 | 备选 1 | 备选 2 |
|------|---------|--------|--------|
| Logo | 从新闻/媒体资源包直接下载 | 从官网 HTML 中提取 SVG/PNG | 从社交媒体头像截图 + 描摹 |
| 产品图片 | 官方图片库 / 新闻稿 | 电商平台高清产品图 | 从官网截图 |
| 色彩 | 从品牌指南 PDF 提取 | 从下载的 logo 中取色 | 从产品摄影中取色 |

**文件组织**：将所有下载的资产保存到项目的 `images/` 目录，使用描述性文件名：
```
images/
├── brand_logo_light.png
├── brand_logo_dark.png
├── brand_product_1.png
├── brand_product_2.png
└── brand_reference_screenshot.png
```

### 步骤 4：验证 + 提取

不要只是提取 HEX 色值——要验证下载的资产确实可用：

| 检查项 | 通过标准 | 不通过时的处理 |
|--------|---------|--------------|
| Logo 分辨率 | PNG 短边 ≥ 200px；SVG 始终通过 | 重新下载或向用户索取 |
| Logo 背景 | 透明背景的 PNG 或 SVG（不是烘焙在彩色背景上的） | 尝试去除背景或向用户索取干净版本 |
| 产品图片质量 | 清晰、光线好、宽度 ≥ 800px | 寻找替代图片或标记为占位符 |
| 色彩准确性 | 提取的 HEX 在 ≥ 2 个来源间一致 | 使用最权威的来源（品牌指南 > 网站 > 社交媒体） |
| 字体识别 | 确认字体名称及许可证状态 | 映射到最接近的系统安全字体替代 |

**取色方法**：
1. 从品牌指南：直接使用标注的色值
2. 从 logo：取色主要填充色
3. 从产品/网站：识别主色、辅色和强调色
4. 始终验证对比度（文字在背景上 ≥ 4.5:1）

### 步骤 5：固化为 brand-spec.md

在项目目录中创建 `brand-spec.md` 文件，使用以下模板：

```markdown
# 品牌规格说明：[品牌名称]

## 品牌身份
- **公司**：[公司全称]
- **行业**：[行业领域]
- **品牌个性**：[2-3 个关键词，如"创新、亲和、高端"]

## Logo
- **主 logo**：`images/brand_logo_light.png`（尺寸：宽x高）
- **深色版本**：`images/brand_logo_dark.png`（如有）
- **安全空间**：[logo 周围最小留白，如"logo 标志字母的高度"]
- **最小尺寸**：[可接受的最小显示尺寸]

## 色彩方案

| 角色 | HEX | 来源 | 用途 |
|------|-----|------|------|
| 主色 | #XXXXXX | [品牌指南 / 从 logo 提取] | 品牌主色，标题，关键元素 |
| 辅色 | #XXXXXX | [来源] | 辅助元素，背景 |

- **品牌字体**：[字体名称] → **PPT 安全映射**：[系统字体]

- **正文字体**：[映射的系统字体]

## 图片资产

| 文件 | 类型 | 尺寸 | 用途 |

| brand_product_1.png | 产品照片 | 宽x高 | 产品展示页 |
| ... | ... | ... | ... |

## 视觉指南
- **设计方向**：[基于收集资产的简要描述]
```

## 素材质量门槛：5-10-2-8 原则

搜索品牌资产或参考材料时：

| 数字 | 含义 |
|------|------|
| **5** | 至少搜索 **5 轮**（不同的查询词、不同的来源） |
| **10** | 至少收集 **10 份**候选材料 |
| **2** | 每个类别选出**最佳的 2 份** |
| **8** | 每份入选材料的质量评分必须 **≥ 8/10** |

**质量评分标准**（每项 1-10 分）：

| 评分维度 | 8 分以上意味着 |
|---------|--------------|
| 分辨率 / 清晰度 | 在目标显示尺寸下清晰锐利，无压缩伪影 |
| 品牌真实性 | 明确来自官方来源，使用当前品牌形象（非过时版本） |
| 可用性 | 透明背景（logo）、正确的宽高比、无水印 |
| 相关性 | 直接支持演示文稿的主题和信息 |

**硬性规则**：CSS 轮廓图、占位矩形、或 AI 生成的品牌 logo **绝不是**真实品牌资产的可接受替代品。如果找不到真实资产，使用虚线边框占位符并告知用户。

## 与八项确认的衔接

完成品牌资产协议后：

| 确认项 | brand-spec.md 如何影响 |
|--------|----------------------|
| **e. 配色方案** | 使用 brand-spec.md 中的颜色，替代通用行业配色 |
| **f. 图标** | 选择图标库时考虑品牌的视觉风格（圆润的品牌 → tabler-filled） |
| **g. 字体排版** | 使用 brand-spec.md 中的 PPT 安全字体映射 |
| **h. 图片** | 品牌产品照片属于"已有"资产；剩余缺口可用 AI 生成 |
| **d. 风格** | 品牌个性关键词指导风格选择 |
